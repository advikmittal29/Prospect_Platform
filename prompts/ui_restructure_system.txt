================================================================================
SYSTEM PROMPT: UI RESTRUCTURING WITH AGENT STUDIO TREE VIEW
================================================================================

OBJECTIVE:
Restructure the prospect platform UI to implement a hierarchical "Agent Studio" 
with a tree view navigation system. Replace flat navigation with a nested, 
agent-centric interface where each agent acts as a container for its specific 
business domains (companies, prospects, lead sources).

NAVIGATION BEHAVIOR:
- Tree view on left sidebar (collapsible per agent)
- Single agent expanded by default (the active agent)
- Clicking agent name toggles expansion
- Clicking child menu items (Dashboard, Companies, Prospects, etc.) navigates immediately
- Dashboard is first menu item for quick access to agent metrics
- Visual indicators for:
  * Active/Selected agent
  * Child page currently being viewed
  * Agent status (active/paused/archived)

SECTION 4: DATA FLOW & STATE MANAGEMENT
================================================================================

TREE NAVIGATION FLOW:
1. User clicks agent in tree вҶ’ Load agent profile + set active
2. User clicks child menu вҶ’ Navigate to agent/:agentId/section
3. User expands/collapses agent вҶ’ Toggle expand state in local storage
4. User edits settings вҶ’ Show unsaved indicator вҶ’ Save to backend

API ENDPOINTS REQUIRED:
GET    /api/agents                    вҶ’ List all agents with status
GET    /api/agents/{agentId}          вҶ’ Full agent profile + settings
POST   /api/agents                    вҶ’ Create new agent
PUT    /api/agents/{agentId}          вҶ’ Update basic info
PUT    /api/agents/{agentId}/profile  вҶ’ Update sales profile
PUT    /api/agents/{agentId}/keywords вҶ’ Update targeting keywords
PUT    /api/agents/{agentId}/policy   вҶ’ Update runtime policies
DELETE /api/agents/{agentId}          вҶ’ Archive/delete agent

FORM STATE MANAGEMENT:
- Use React Hook Form for complex forms
- Implement optimistic updates with rollback on error
- Persist form drafts to localStorage
- Show conflict warning if backend updated while editing
